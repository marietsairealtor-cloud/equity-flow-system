# GOVERNANCE_CHANGE_PR104.md

## What changed

Build Route v2.4 Item 9.4 token format validation. Migration 20260311000000 drops and recreates lookup_share_token_v1 with a format guard that fires before hashing. Guard enforces shr_ prefix, 64-char lowercase hex body, minimum length 68. All eight existing pgTAP test files updated to use shr_-prefixed tokens matching the new format rules. New test file 9_4_token_format_validation.test.sql added with 9 tests.

## Why safe

Signature of lookup_share_token_v1(text, uuid) is unchanged. The format guard only adds an early-exit path for tokens that could never have matched a stored hash (stored hashes are always of shr_-prefixed tokens per 8.8 contract). Valid callers using tokens generated by create_share_token_v1 are unaffected. All prior tests updated atomically in the same PR. No RLS, privilege, or tenancy logic changed.

## Risk

Low. The only behavioral change is that malformed tokens (no shr_ prefix, wrong length, wrong charset) now return NOT_FOUND before reaching the digest call instead of after a hash miss. Response shape is identical. No caller observes a different code or message for any token that was previously valid.

## Rollback

Revert migration 20260311000000_9_4_token_format_validation.sql via a new forward migration that drops and recreates the prior function body from 20260310000003. Restore prior token strings in all affected test files via git revert of this PR. No data migration required.
